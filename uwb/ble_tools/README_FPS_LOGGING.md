# UWB 测距 - FPS 计算与日志说明

## 📊 FPS 计算逻辑

系统使用 **3 种不同的 FPS 计算方式**，分别用于不同的分析目的：

### 1. **实时 FPS** (Realtime FPS)
- **位置**: `RangingStats.fps()` (第72-79行)
- **计算方法**: 基于滑动窗口（最近100帧）
- **公式**: 
  ```python
  实时FPS = (窗口内帧数 - 1) / (最后时间戳 - 第一个时间戳)
  ```
- **用途**: 反映**当前瞬时**的帧率，能快速响应帧率变化
- **特点**: 
  - 窗口大小可配置（默认100帧）
  - 对短期波动敏感
  - 适合实时监控

### 2. **周期 FPS** (Period FPS)
- **位置**: `get_summary_and_reset()` (第145行)
- **计算方法**: 两次周期统计之间的平均帧率
- **公式**:
  ```python
  周期FPS = 周期内收到的帧数 / 周期时长
  ```
- **用途**: 反映**每个统计周期**（默认3秒）的平均帧率
- **特点**:
  - 每次统计后重置计数器
  - 适合观察不同时间段的性能差异
  - 周期可通过 `--summary-interval` 参数配置

### 3. **全局 FPS** (Global FPS)
- **位置**: `RangingStats.global_fps()` (第86-90行)
- **计算方法**: 从程序启动到当前的总体平均帧率
- **公式**:
  ```python
  全局FPS = 总帧数 / 总运行时间
  ```
- **用途**: 反映**整个测试会话**的平均性能
- **特点**:
  - 不受短期波动影响
  - 适合评估整体性能
  - 用于最终统计报告

## 📝 日志系统

### 日志文件结构

每次运行会生成 **2 个文件**：

1. **CSV 数据文件** (`uwb_log_YYYYMMDD_HHMMSS.csv`)
   - 逐帧记录所有测距数据
   - 包含字段：
     - `timestamp`: Unix 时间戳
     - `elapsed_s`: 相对开始时间（秒）
     - `frame_no`: 帧序号
     - `distance_m`: 距离（米）
     - `angle_deg`: 角度（度）
     - `pitch_deg`: 俯仰角（度）
     - `fps`: 实时FPS
     - `frame_gap_ms`: 帧间隔（毫秒）
     - `rssi`: RSSI 信号强度值

2. **文本日志文件** (`uwb_log_YYYYMMDD_HHMMSS.log`) ✨ **新增**
   - 记录所有周期统计摘要
   - 记录最终统计摘要
   - 包含时间戳和完整的性能指标

## 🎯 使用场景

### 场景 1: 实时监控
- 查看终端输出的**实时FPS**
- 观察每帧的 `gap` 值检测丢帧

### 场景 2: 周期性能分析
- 查看每3秒的**周期统计**
- 对比不同时间段的性能差异
- 检查 `.log` 文件中的周期摘要

### 场景 3: 离线数据分析
- 使用 CSV 文件进行详细分析
- 运行 `analyze_uwb_logs.py` 生成可视化图表
- 对比不同测试场景的性能

### 场景 4: 长期性能评估
- 查看**全局FPS**和最终统计摘要
- 评估整体稳定性和可靠性
- 检查 `.log` 文件末尾的最终摘要

## ⚙️ 配置参数

### 命令行参数

```bash
sudo python3 run_uwb_ble.py \
  --port /dev/ttyUSB0 \           # 串口设备
  --baudrate 115200 \             # 波特率
  --summary-interval 3.0 \        # 周期统计间隔（秒）
  --csv-path /custom/path.csv \   # 自定义CSV路径
  --no-csv                        # 禁用CSV日志
```

### 代码配置

在 `SerialHandlerStandalone.py` 中：

```python
# 滑动窗口大小（影响实时FPS计算）
self.stats = RangingStats(window_size=100)  # 默认100帧

# 丢帧检测阈值
if gap > avg_gap * 2.5:  # 间隔超过平均值2.5倍视为丢帧
    self.drop_count += 1
```

## 🔍 FPS 对比示例

假设在一次测试中：
- 前30秒稳定在 6Hz
- 30-60秒出现波动，降到 4-5Hz
- 60-90秒恢复到 6Hz

**3种FPS的表现**：
- **实时FPS**: 会快速反映每个阶段的变化 (6.0 → 4.5 → 6.0)
- **周期FPS**: 每3秒报告一次，显示该周期的平均值
- **全局FPS**: 始终显示整体平均，约 5.5Hz

## 🐛 常见问题

### Q: 为什么有3种FPS？
A: 不同的FPS用于不同的分析目的：
- 实时FPS：监控当前状态
- 周期FPS：对比不同时间段
- 全局FPS：评估整体性能

### Q: 丢帧是如何判断的？
A: 当帧间隔 > 平均帧间隔 × 2.5 时，计为一次丢帧。

### Q: CSV 文件中的 FPS 是哪一种？
A: CSV 中记录的是**实时FPS**，因为它最能反映每一帧的瞬时状态。

### Q: 如何禁用日志？
A: 使用 `--no-csv` 参数：
```bash
sudo python3 run_uwb_ble.py --port /dev/ttyUSB0 --no-csv
```

### Q: 日志文件太大怎么办？
A: 可以定期清理旧日志，或使用自定义路径：
```bash
sudo python3 run_uwb_ble.py --csv-path /tmp/uwb_test.csv
```
