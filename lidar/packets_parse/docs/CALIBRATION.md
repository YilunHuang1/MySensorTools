# 角度校准详解

## 核心概念

**校准参数**: 修正每个激光通道的硬件偏差，使点云坐标精确

```
原始数据 (含硬件偏差)
    ↓ [应用校准表]
精确的 3D 坐标
```

---

## 文件格式

**文件**: `config/calibration/Vanjee_722z_VA.csv`

```csv
-0.25,-2.423793797      # CH0:  垂直 -0.25°,  水平偏差 -2.424°
2.45,1.567974548        # CH1:  垂直  2.45°,  水平偏差 +1.568°
5.15,-2.476401604       # CH2:  垂直  5.15°,  水平偏差 -2.476°
...
40.25,1.671322722       # CH15: 垂直 40.25°,  水平偏差 +1.671°
```

**格式**:
- 第1列: 垂直角度 (度)
- 第2列: 水平角度偏差 (度)
- 16 行 (16 个通道)

---

## 参数的来源

**出厂校准流程** (万集执行):

1. 用标准反射目标扫描雷达
2. 对比 **实际点云位置** vs **理论设计位置**
3. 计算偏差: $\Delta\theta = \theta_{measured} - \theta_{designed}$
4. 保存到 CSV 文件

**为什么需要校准**:

| 因素 | 影响 |
|------|------|
| 镜头安装公差 | ±0.5mm 的偏差 → 角度误差 2-3° |
| 棱镜分光偏心 | 左右两侧光路不对称 |
| 温度漂移 | 光学元件热膨胀系数差异 |
| 旋转平台不平衡 | 旋转时的动态偏差 |

| 型号 | 通道数 | 水平偏差范围 | 精度等级 |
|------|-----|---------|----|
| **WLR-722Z** (你的) | 16 | -3.42° ~ +1.67° | 🏆 **高精度** |
| WLR-722 | 32 | -1.52° ~ +2.33° | 🏆 **高精度** |
| WLR-722F | 16 | -2.85° ~ +1.57° | 🏆 **高精度** |
| WLR-722H | 16 | -2.85° ~ +1.57° | 🏆 **高精度** |
| WLR-720-16 (老款) | 16 | 0° (全是 0) | ⚠ **低精度** |
| WLR-721-64 | 64 | -7.78° ~ +5.09° | 🏆 **高精度** |

---

## 奇偶通道模式

**WLR-722Z 的特征**:

```
奇数通道 (CH1,3,5,...):  +1.56° ~ +1.67° ← 一侧
偶数通道 (CH0,2,4,...): -2.42° ~ -3.42° ← 另一侧
```

**含义**: 16 个通道分别安装在镜头两侧，采用棱镜分光结构

## 代码中的应用

### 1. 加载校准参数

```python
from src.core.calibration import CalibrationManager

calib = CalibrationManager()
calib.load_default('722z')  # 自动查找 config/calibration/Vanjee_722z_VA.csv
vert_angles, horiz_angles = calib.get_all_angles()
```

### 2. 坐标变换中使用

```python
# 在每个点的 3D 坐标计算时应用
angle_vert = vert_angles[chan]                    # 第 chan 通道的垂直角度
angle_horiz_final = (horiz_angles[chan] +         # 水平偏差
                     azimuth * 10 + 360000) % 360000

# 最终坐标计算
xy = distance * cos(angle_vert)
x = xy * sin(angle_horiz_final) + ...
y = xy * cos(angle_horiz_final) + ...
z = distance * sin(angle_vert) + ...
```

**在 10 米处的点**:

| 场景 | 角度误差 | 距离误差 |
|------|--------|--------|

→ 校准参数让精度提升 **400 倍!**

## 常见问题

### Q: 是不是每颗雷达都不一样?

**A**: 分情况讨论

- **型号级**: ✓ 大体一致，所有 WLR-722Z 使用同一个文件
- **批次级**: ⚠ 可能有微小差异 (< 0.1°)，但通常用同一个文件
- **单台级**: ❌ 极少进行逐台标定 (成本太高)

### Q: 如果校准文件丢失怎么办?

**A**: 三个方案

```python
# 方案 1: 使用其他型号的参数 (精度降低 2-3mm)
calib = CalibrationManager()
calib.load_default('722')  # 使用 722 的参数作为替代

# 方案 2: 完全不做校准 (精度降低 5mm+)
vert_angles = [int(v * 1000) for v in [-0.25, 2.45, ...]]
horiz_angles = [0] * 16  # 全是 0，不做水平校准

# 方案 3: 重新标定 (需要标定工具)
# 联系万集获取标定工具
```

### Q: 如何验证校准参数正确?

**A**: 运行分析脚本

```bash
python src/utils/analyze_calibration.py
# 输出: 各型号的校准参数对比表
```
